    ▸ ML MODEL SPECIFICATIONS
    Anomaly Detection: Z-Score method applied to cycle time series. Points exceeding ±2σ flagged as anomalies. Threshold tunable based on process tolerance. Implementation: client-side JavaScript; production: Python scikit-learn.
    Bottleneck Prediction: Ratio of actual cycle time to ideal cycle time per station. A Random Forest classifier (suggested) would use features: CT variance, alarm frequency, idle spikes to predict fault events 5–10 min ahead.
    LSTM Forecasting: Sequence model on OEE rolling windows for next-period prediction. Requires ≥2 weeks of historical data. Current session (22 min) used for trend extrapolation via linear regression.

    ▸ DIGITAL THREAD DESIGN NOTES
    OT/IT Integration: Rockwell PLCs expose tags via OPC-UA server. An edge gateway (e.g., Ignition Edge, Node-RED) subscribes to production tags and publishes to MQTT broker or writes directly to a time-series database like InfluxDB or TimescaleDB.
    Data Governance: Each CSV file is named with PLC IP + process name + Unix timestamp, enabling automatic lineage tracing. The _KPI_sorted.csv files document which tags changed between sessions — critical for schema evolution handling.
    ML Pipeline: Python script reads from DB → extracts features (CT mean, CT σ, alarm frequency) → feeds Random Forest classifier for fault prediction. LSTM model trained on ≥2 weeks of data for OEE sequence forecasting.
